# task:

Реализуем систему для выполнения миграции данных.
В общем случае миграция выполняется из источника в приёмник, 
при этом данные в процессе проходят через набор преобразований. 
Преобразования должны быть настраиваемыми, их можно комбинировать, 
менять порядок, добавлять новые — и при этом ядро вашей системы менять нельзя. 
Ваша задача — сделать такую архитектуру, 
чтобы поведение системы можно было менять без изменения реализации 
(только за счёт подключения других модулей).

- ввод 
`10; Ivan   Petrov ; ivan@example.com`
- вывод
`{"id":<id>,"firstName":"<firstName>","lastName":"<LASTNAME>","email":"<email>"}`

# Реализация:

`datamigrate\client\DataPool.java` - клиент слушает данные
`datamigrate\dto` - папка дтошек
`dto\AccountDTO.java` - рекорд аккаунта
`datamigrate\parser` - парсер и все для него
`parser\exception` - эксепшены парсера
`exception\ParseException.java` - главный эксепшен парсера

`datamigrate\parser\ParserInterface.java` - интерфейс парсеров
`parser\AccountParser.java` - парсер для аккаунта
